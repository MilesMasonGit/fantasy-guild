# Auditor Skill

Use this skill when performing a systematic "deep clean" or optimization pass on the Fantasy Guild Idle codebase. This skill enforces a strict, human-in-the-loop protocol to ensure feature integrity while removing AI bloat.

## When to Use
- During the dedicated "Optimization and Codebase Cleaning" phase.
- When a module has become too complex or contains obvious AI-generated redundancy.
- When migrating legacy Vanilla logic to the modern React architecture.

## The 6-Step Audit Protocol
1. **Step 1: Orientation & Trace**: Read [Codebase Map](file:///c:/Users/16048/.gemini/antigravity/brain/bdf68d8f-ef61-449c-bf98-5b6893241cef/codebase_map.md) and [Audit Log](file:///c:/Users/16048/.gemini/antigravity/brain/bdf68d8f-ef61-449c-bf98-5b6893241cef/audit_log.md). Map dependencies.
2. **Step 2: Intent Reflection**: Present your understanding to the USER. **HALT for approval.**
3. **Step 3: Evolution Blueprint**: Present the proposed "Modern Pattern" and bloat reduction goals. **HALT for approval.**
4. **Step 4: Structural Evolution**: Execute the refactor using [Golden Standard](file:///c:/Users/16048/.gemini/antigravity/brain/bdf68d8f-ef61-449c-bf98-5b6893241cef/standard_component.js).
5. **Step 5: De-Bloat Review**: Perform an internal check for any *new* AI complexity introduced.
6. **Step 6: Update Logs**: Record changes in [Audit Registry](file:///c:/Users/16048/.gemini/antigravity/brain/bdf68d8f-ef61-449c-bf98-5b6893241cef/audit_registry.md) and session logs.

## Quality Constraints
- **Max Length**: No file > 200 lines (registries excepted).
- **Patterns**: Use ES6+ idiomatic JS. Avoid "comment-heavy" or "branch-heavy" AI logic.
- **DRY**: Actively consolidate repeated patterns across files.
